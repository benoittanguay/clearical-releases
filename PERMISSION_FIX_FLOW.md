## Detection Logic Pseudocode

```python
def check_screen_permission():
    # Step 1: Check TCC database
    status = systemPreferences.getMediaAccessStatus('screen')

    if status == 'not-determined':
        # Trigger permission prompt
        desktopCapturer.getSources(['screen'])
        return systemPreferences.getMediaAccessStatus('screen')

    if status == 'granted':
        # Step 2: VERIFY it actually works (NEW!)
        test_result = testScreenRecordingWorks()

        if not test_result.works:
            # STALE permission detected!
            console.error("STALE PERMISSION DETECTED")
            return 'stale'  # Special status

    return status  # 'granted', 'denied', or 'restricted'


def testScreenRecordingWorks():
    try:
        # Attempt actual screen capture
        sources = desktopCapturer.getSources({
            types: ['screen'],
            thumbnailSize: { width: 100, height: 100 }
        })

        if sources.length == 0:
            return { works: False, error: 'no_sources' }

        thumbnail = sources[0].thumbnail
        size = thumbnail.getSize()

        if size.width == 0 or size.height == 0:
            return { works: False, error: 'empty_thumbnail' }

        # Success!
        return { works: True }

    except error:
        return { works: False, error: str(error) }
```

## Key Insights

### Why This Works

1. **Ground Truth Testing**
   - `systemPreferences.getMediaAccessStatus()` only checks TCC database
   - `desktopCapturer.getSources()` actually tries to capture
   - If TCC says "granted" but capture fails â†’ STALE!

2. **User-Friendly Recovery**
   - Native dialog (not web UI) = more trustworthy
   - Two options (GUI and CLI) = flexibility
   - Clear explanation = no confusion
   - Auto-detection of fix = immediate feedback

3. **Performance Conscious**
   - Small 100x100 thumbnail = fast
   - Only tests when checking permission = not continuous
   - <100ms typical execution time

4. **Security Respecting**
   - No attempt to modify TCC programmatically
   - User must manually grant permission
   - Follows Apple's intentional design

### Why Previous Approaches Fail

âŒ **Only check TCC database**: Shows "granted" even when stale
âŒ **Auto-retry capture**: Silently fails, confuses user
âŒ **Force permission prompt**: Can't re-prompt if already granted
âŒ **Modify TCC directly**: No API exists, would require sudo

âœ… **Our approach**: Detect + Guide = Self-service resolution
